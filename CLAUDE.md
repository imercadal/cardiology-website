## Project Overview

Professional website for Dr. Enrique Mercadal Calaf (cardiologist). Built with Next.js 15 using the App Router. Single-page landing site with contact info, hours, and appointment links.

## Commands

```bash
npm run dev      # Start dev server at http://localhost:3000
npm run build    # Production build
npm run start    # Run production server
npm run lint     # Run ESLint
```

Docker alternative:
```bash
docker-compose up  # Builds and runs dev server on port 3000 with hot reload
```

## Architecture

- **Framework**: Next.js 15 (App Router) with React 19
- **Styling**: Tailwind CSS 3 + custom classes in `globals.css`
- **Animations**: CSS-only (`fadeSlideUp` keyframe in `globals.css`) for cards; Framer Motion used for scroll-driven effects (`FadeOut` component)
- **Icons**: `lucide-react` is installed but currently unused. Always prefer SVG and dont use emojis unless specified.

### Directory Structure

```
app/
  layout.jsx        # Root layout with metadata (full SEO: OG, Twitter, keywords, canonical) and Google Font declarations
  page.jsx          # Main landing page (all content sections) + JSON-LD structured data (Person + MedicalBusiness)
  sitemap.js        # Next.js 15 sitemap — auto-generates /sitemap.xml; reads NEXT_PUBLIC_SITE_URL
  robots.js         # Next.js 15 robots — auto-generates /robots.txt; reads NEXT_PUBLIC_SITE_URL
  globals.css       # Tailwind directives + global element styles (h1, h3, h5, p, button)
                    # + utility classes (.section, .card, .rectangulo)
                    # + accessibility styles (focus-visible, prefers-reduced-motion)
  components/
    RectangleCard.jsx    # Animated card with CSS fadeSlideUp animation; supports ariaLabel, isExternal, delay props
    CopyLinkButton.jsx   # Clipboard copy button with visual feedback ('use client')
    FadeOut.jsx          # Scroll-driven fade-out wrapper using Framer Motion useScroll + useTransform; named export { FadeOut }; respects prefers-reduced-motion
    ScrollLines.jsx      # Decorative fixed lines that move on scroll (vertical slides right, horizontal slides up); CSS-only fallback for prefers-reduced-motion
public/             # Static assets (images, SVGs)
```

### Key Patterns

- Components using browser APIs or interactivity require `'use client'` directive
- Custom Tailwind colors (blues/reds) defined in `tailwind.config.mjs`
- Font CSS variables set in `layout.jsx` and consumed via `globals.css`
- Files use `.jsx` extension (TypeScript is configured but not used in components)
- Heading hierarchy: `h1` (section titles), `h2` (subtitle — no global style, styled inline), `h3` (subsection headings via global style), `h5` (card labels)
- `RectangleCard` accepts optional `ariaLabel` prop forwarded to the underlying `<a>` or `<Link>`
- WCAG 2.1 AA: `focus-visible` outlines on all interactive elements, `aria-live` on copy button, `@media (prefers-reduced-motion)` disables card animations and scroll effects
- Framer Motion's `useReducedMotion()` used in `FadeOut` to disable scroll-driven opacity when reduced motion is preferred
- Global `button` styles defined in `globals.css`; standalone `<a>` buttons styled inline with equivalent Tailwind classes

### SEO

- Full metadata in `layout.jsx`: title template, description, keywords, canonical, Open Graph (`es_CL` locale), Twitter card
- `metadataBase` set from `NEXT_PUBLIC_SITE_URL` env var (fallback: `https://drmercadal.cl`) — **must be set in production**
- JSON-LD `@graph` in `page.jsx` with `Person` (doctor) and `MedicalBusiness` (clinic) schemas
- `/sitemap.xml` and `/robots.txt` auto-generated by Next.js from `app/sitemap.js` and `app/robots.js`
- When adding new routes, add them to `sitemap.js`
